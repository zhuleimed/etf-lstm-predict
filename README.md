## 💡 核心理念

> **用深度学习预测 ETF 次日方向，以交易成本为阈值生成买卖信号，对比 buy-and-hold 基准**

### 为什么用 LSTM-Transformer？

| 组件 | 作用 |
|------|------|
| **双向 LSTM** | 捕捉价格序列的短期和长期依赖 |
| **多头注意力** | 关注不同时间步的重要特征 |
| **残差连接** | 稳定深层网络训练 |
| **早停机制** | 防止过拟合（数据有限时至关重要） |

### 为什么选 ETF 而不是个股？

- ETF 波动更平稳，预测信噪比更高
- 4 只沪深300 ETF 高度相关，可相互验证信号一致性
- ETF 免印花税，交易成本更低

## 🔄 每日执行流程

```
每日 cron 21:00 触发
  │
  ├─ 🛡 防重复运行检测（杀残留进程）
  │
  ├─ 📅 交易日判断（baostock API query_trade_dates）
  │   └─ 非交易日 → 静默退出
  │
  ├─ 📡 加载 4 只 ETF 日线数据（baostock, 45s 超时保护）
  │
  ├─ 🤖 加载 LSTM 模型 → 预测次日涨跌幅
  │   └─ 模型过期（>5 天）→ 自动重训练
  │
  ├─ 📊 阈值判断 → 生成买卖信号
  │   ├─ 🟢 >+0.36% → 买入
  │   ├─ 🔴 <-0.36% → 卖出
  │   └─ ⚪ 其余 → 持有
  │
  ├─ 💹 执行昨日待处理订单（以开盘价成交）
  │
  ├─ 📈 计算策略 vs 基准（buy-and-hold）收益率
  │
  ├─ 📱 WxPusher 推送日报到微信
  │
  └─ 🧹 退出时清理 PID 锁文件
```

## 📊 信号逻辑

### 涨跌标准

```
预测次日涨幅 > +0.36%  → 🟢 买入（下一交易日开盘）
预测次日跌幅 < -0.36%  → 🔴 卖出（下一交易日开盘）
其余情况               → ⚪ 持有不动
```

### 阈值设计

| 成本项 | 比例 | 说明 |
|--------|------|------|
| 滑点 | 0.30% | 买卖价差 + 冲击成本 |
| 佣金 | 0.05% | 最低 5 元 |
| 其他 | 0.01% | 预留 |
| **合计** | **0.36%** | 覆盖交易成本后的最小盈利要求 |

### 基准对比

- **策略收益**：模拟盘组合的实际收益率
- **基准收益**：等权买入持有全部 4 只 ETF 的收益率
- **超额收益** = 策略收益 − 基准收益

## 🧠 模型架构

### 简化版 LSTM（默认，快速版）

```
输入(30天×32维) → LSTM(32) → 全连接(16) → 输出(次日涨跌幅)
```

- 训练时间：约 2~3 分钟/ETF
- 方向准确率：~55-60%

### 增强版 LSTM-Transformer（完整版）

```
输入 → 适配层 → 双向LSTM → Transformer编码器(残差) → 全连接 → 输出
```

- 训练时间：约 5~8 分钟/ETF
- 方向准确率：~55-65%

### 特征体系

| 类别 | 特征 | 维度 |
|------|------|------|
| 基础 | 收益率、对数收益、振幅 | 4 |
| 移动平均 | MA5/10/20 及其偏离度 | 6 |
| 动量 | RSI6/12, MACD, 布林带 | 9 |
| 波动 | ATR, 波动率 | 3 |
| 成交量 | 量比、额比 | 4 |
| 价格动量 | 3/5/10 日动量 | 3 |
| **合计** | | **~32 维** |

## 🔬 关键技术细节

- **消除 look-ahead bias**：信号基于当日收盘数据计算，次日开盘执行
- **baostock 超时保护**：每只 ETF 的 API 调用设 45 秒超时，防止网络挂起
- **防重复运行**：启动时扫描 `/proc` 精确识别残留进程并清理
- **PID 锁文件**：退出时自动删除，防止并发冲突
- **原子写入**：state.json 先写 `.tmp` 再 rename
- **ETF 免印花税**：交易成本仅含佣金 + 滑点
- **特征精简**：为适配 baostock ETF 数据量（~96 行），去掉长窗口特征

## ❓ 常见问题

### Q1: 模型方向准确率只有 60%，够用吗？

金融时间序列预测中，>55% 的方向准确率已具有交易价值。60% 准确率 × 0.36% 平均收益率 × 250 交易日 ≈ 年化 15%+ 的毛收益（扣除成本前）。

### Q2: 为什么只用 96 行数据训练？

baostock 的 ETF 日线数据从 2026-01 开始提供（约 96 行）。对于 LSTM 模型，96 行 × 60% 训练集 = 57 个样本，配合早停和 dropout，仍可训练出有意义的模型。

### Q3: 模型多久重训练一次？

每 5 个交易日自动重训练。重训练时使用截至当日的全部数据（滚动增加）。

### Q4: 基准收益 2.15% 怎么来的？

从数据起始日（2026-01-05）起等权买入持有全部 4 只 ETF 的累计收益率。

### Q5: 首次运行需要多久？

首次需要训练 4 个模型，约 8~12 分钟。之后每日运行仅需加载模型做预测，约 5~10 秒。
